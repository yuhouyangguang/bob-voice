# BOB Voice Backend 接口测试文档

版本：v1.2  
日期：2026-06-05  
依据：[API.md](API.md)

## 1. 测试目标

验证 BOB Voice 后端以下能力：

- 用户登录、JWT 和权限控制
- 音频转写与文本加工任务
- 任务状态、重试、取消和软删除
- 分片上传和断点续传
- 转写结果查询、编辑和说话人修改
- 术语纠错、用户管理、督办清单和领导档案库
- Markdown、Word、JSON 和 ZIP 导出
- Socket.IO 任务进度和实时语音转写
- 异常参数、越权访问和文件限制

## 2. 测试范围

### 2.1 本轮包含

| 模块 | 测试范围 |
|---|---|
| 健康检查 | 服务状态与 ASR provider |
| 认证 | 登录、登出、刷新、账号锁定 |
| 任务 | 创建、查询、筛选、取消、重试、删除 |
| 转写结果 | 三种视图、文本编辑、说话人修改 |
| 上传 | 初始化、分片、状态、合并 |
| 文档 | Markdown、Word、JSON、ZIP |
| 督办清单 | 生成、查询、编辑 |
| 档案库 | 自动归档、多词搜索、多选筛选、摘要高亮、领导详情、历史发言 |
| 管理后台 | 纠正规则 CRUD、筛选、导入导出、用户创建编辑、停用、重置密码、解锁、统计 |
| Socket.IO | 任务进度、实时 PCM 转写 |
| 安全 | 未认证、跨用户、普通用户访问管理接口 |

### 2.2 本轮不包含

- LDAP、企业微信 OAuth2.0
- 声纹注册和自动说话人识别
- Celery/Redis 多节点任务恢复
- MySQL、MinIO 集群验证
- 病毒扫描、HTTPS 和内网 CA
- 500MB 以上文件

## 5. 执行策略

| 测试类型 | 执行时机 |
|---|---|
| pytest 自动化回归 | 每次提交后 |
| HTTP 冒烟测试 | 每次部署后 |
| DashScope 真实联调 | ASR 配置或模型变更后 |
| 文件兼容测试 | 上传、转码逻辑变更后 |
| Socket.IO 测试 | 实时协议或鉴权变更后 |
| 权限和越权测试 | 认证、资源查询逻辑变更后 |
| 档案库回归 | 档案模型、搜索、说话人标注或数据库迁移变更后 |

优先级：

- `P0`：主流程或安全关键，发布前必须通过
- `P1`：重要功能，发布前应通过
- `P2`：边界或兼容性测试

## 6. 通用检查项

每个 HTTP 用例除业务预期外，还应检查：

1. HTTP 状态码正确。
2. JSON 响应 `Content-Type` 正确。
3. 错误响应包含 `error` 和 `message`。
4. 响应中不包含密码、密码哈希、API Key 和本地存储路径。
5. 当前用户只能访问自己的任务和上传记录。
6. 文件下载名称、扩展名和 MIME 类型正确。
7. 中文内容为 UTF-8，无乱码。
8. 服务日志无未处理异常和密钥泄漏。

## 8. 主流程验收场景

### 8.1 文本加工主流程

1. 管理员登录并新增 `封控→风控` 规则。
2. 普通用户上传 `sample.txt`。
3. 创建文本任务，填写主题和重点发言人。
4. 轮询或订阅任务进度，等待 `completed`。
5. 查询时间轴结果。
6. 确认时间戳和通用说话人标签被清理。
7. 确认“封控”已改为“风控”。
8. 修改段落文本和说话人。
9. 查询或生成督办清单。
10. 下载 Markdown、Word、JSON 和 ZIP。
11. 使用关键词在档案库中检索该段落。
12. 查询领导列表，确认重点发言人已自动生成领导档案。
13. 查询领导详情和历史发言，确认关联会议及发言内容正确。

通过标准：步骤全部成功，数据在各接口间保持一致。

### 8.2 录音转文字主流程

1. 配置有效 `DASHSCOPE_API_KEY`。
2. 上传 `short.wav` 创建音频任务。
3. 订阅任务进度。
4. 确认出现预处理、转写、后处理阶段。
5. 等待任务完成。
6. 检查段落时间戳、文字和置信度。
7. 下载 Word 和 ZIP。

通过标准：有效中文录音产生非空文本，任务无未处理异常。

### 8.3 实时转写主流程

1. 使用有效用户建立 `/realtime` 连接。
2. 发送 `realtime:start`。
3. 收到 `realtime:ready`。
4. 按 100ms 一帧发送 `realtime.pcm`。
5. 收到 `realtime:result` 中间结果。
6. 至少收到一条 `is_final=true` 的最终结果。
7. 发送 `realtime:stop`。

通过标准：识别文字非空、事件顺序合理、连接间结果隔离。

### 8.4 档案库主流程

1. 使用 `user001` 创建“科技金融专题调研”文本任务。
2. 设置会议时间、地点、会议类型和重点发言人“李主任”。

7. 检查摘要、关键词高亮、地点、文档地址等字段。
8. 查询 `/library/leaders/{id}`，检查档案和关联会议。
9. 查询 `/library/leaders/{id}/speeches`，检查完整发言和分页。
